# [TagName] Description 
# 위 형식에 맞춰 커밋 메세지를 작성
# - Description은 50자 이내로 무엇을 했는지 작성
# - 마침표로 끝내지 않음 ~ 이 줄 바로 아래 공백줄에 작성

# 이 줄 바로 아래 공백은 지우지 마세요 (제목과 본문의 분리를 위함, 아무것도 적지도 말 것)

################
# Detail
# 위 형식에 맞춰 본문(구체적인 내용 => 무엇을, 왜 했는지)을 작성, 제목과 일치하면 생략가능. 
# 여러 줄의 메시지를 작성할 땐 "-"로 구분 (한 줄은 72자 이내) ~ 이 줄 바로 아래 공백줄에 작성

################
# Footer
# 꼬릿말을 작성. 생략가능
# example.1 => Related Issue: #123
# example.2 => Closes #456  ~ 이 줄 바로 아래 공백줄에 작성

################
# 사용 가능한 Tag 목록
# feat    : 새로운 기능 추가
# fix     : 버그 수정
# docs    : 문서 수정
# test    : 테스트 코드 추가
# refact  : 코드 리팩토링
# style   : 코드 의미에 영향을 주지 않는 변경사항
# chore   : 빌드 부분 혹은 패키지 매니저 수정사항
